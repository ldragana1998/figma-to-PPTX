? Troubleshooting

The Copy button doesn't seem to work

Browsers block clipboard access on files opened directly (file://). The button now falls back automatically, but if it still fails: click inside the command box, select all the text with Ctrl+A, then Ctrl+C.

Access to the path '...\System32\figma.json' is denied

PowerShell opened in C:\WINDOWS\System32, which is write-protected. Run cd C:\Users\Korisnik\Downloads\Calude first, then paste the command again. The Figma call itself already worked — only the file write failed.

{"params":["file_key"],"query":["ids"]} are required (400)

The command got split across lines on paste, so it ran before the variables were set. Copy the command again and paste it as a single line (these commands are one line on purpose).

App says "Loaded 0 embedded images"

You likely have a leftover empty images.json from a failed run. Delete it and re-run the image command:

Remove-Item images.json -ErrorAction SilentlyContinue

The command prints Figma returned N URL(s)… and Wrote images.json with N image(s). so you can see the real count.

It shows a huge number of images (e.g. 248)

That count is per selection. Select one frame at a time — the image command only fetches photos inside the frames you tick. 60+ in one selection may exceed Canva's 50 MB import limit.

403 Forbidden from Figma

The token is wrong, expired, or missing the File content (read) scope. Generate a fresh token and rebuild the command. 404 instead means the file link/key is wrong.

Where do the files go?

Both figma.json and images.json are written into whatever folder PowerShell is in — so always cd to C:\Users\Korisnik\Downloads\Calude first. That keeps them next to this app.
